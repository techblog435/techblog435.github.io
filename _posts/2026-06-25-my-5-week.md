## What is a Random Forest?

A **Random Forest** is a powerful supervised machine learning algorithm used for both **Classification** (predicting a label or category) and **Regression** (predicting a continuous numerical value). 

It belongs to a class of architecture known as **Ensemble Learning**, which combines the predictions of multiple base models to generate a single, highly accurate, and robust output. Instead of relying on a single decision maker, it builds a "forest" of independent **Decision Trees**.

---

## Core Mechanics: How the Forest is Built

A Random Forest achieves its high accuracy and robustness by introducing randomness into the training process. This is done through two core techniques:

### 1. Bootstrap Aggregating (Bagging)
If every tree in the forest trained on the exact same dataset, they would all look identical and make the same mistakes. To prevent this, Random Forest uses **Bagging**. 
* Given a dataset with $N$ rows, the algorithm creates multiple subsets by sampling the rows **with replacement**. 
* This means some rows will be repeated in a subset, while others are left out entirely. 
* Each individual decision tree is trained on a completely different subset of data.

### 2. Feature Randomness (The Random Subspace Method)
In a standard decision tree, the model looks at every single available feature (column) and picks the absolute best one to split the data. Random Forest disrupts this:
* When splitting a node, each tree is only allowed to choose from a **random subset of features** rather than the entire pool.
* If a dataset has 20 features, a tree might only be allowed to look at 4 random features at any given split.
* This forces the trees to become highly diverse and prevents a single, overly dominant feature from dictating the structure of every single tree.

---

## Aggregating the Results

Once the forest is trained and you feed it new, unseen data, every single tree casts its own independent prediction. The algorithm aggregates these individual decisions to get the final answer:

| Task Type | Aggregation Method | How it Works |
| :--- | :--- | :--- |
| **Classification** | **Majority Voting** | If 70 trees predict "Class A" and 30 trees predict "Class B", the final model output is **Class A**. |
| **Regression** | **Averaging** | If the model is predicting a house price, it takes the mathematical average of the numbers predicted by all 100 trees. |

---


* **Highly Resistant to Overfitting:** While a single decision tree easily overfits by memorizing training noise, the averaging mechanism of a forest cancels out individual errors, making it exceptionally robust.
* **Handles Missing Values:** It can maintain excellent accuracy even when a significant portion of the data is missing.
* **Feature Importance:** Out of the box, Random Forest calculates how much each variable contributed to the final prediction, making it highly interpretable for data exploration.
