## SQL Aggregations: Utilizing GROUP BY, HAVING, and summary functions

In database engineering, real power lies in summarizing vast collections of records to find meaningful totals. **SQL Aggregations** allow us to perform mathematical operations on a set of values to return a single summarized value. 

Using functions like `SUM`, `AVG`, and `COUNT`, we can group raw information into structured categories using the `GROUP BY` clause. If we want to filter these summarized groups based on specific conditions, we use the `HAVING` clause, which acts like a `WHERE` clause but specifically for aggregated data.

```sql
-- Calculating the average lab score for each course, filtering out low averages
SELECT CourseName, COUNT(StudentID) AS TotalStudents, AVG(LabScore) AS AverageScore
FROM StudentMarks
GROUP BY CourseName
HAVING AVG(LabScore) >= 70;

```
